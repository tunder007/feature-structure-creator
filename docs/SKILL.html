Feature-structure creator

Human operator guide. Scaffolds a new feature dossier on the proven lumen-features 8-section schema — a dense <slug>.md (AI source of truth) plus a generated <slug>.html (human view) in a numbered NN-<slug>/ folder — and keeps FEATURES.md in sync. Where the Deterministic Checker measures a catalog's uniformity, this skill produces dossiers that pass that check by construction.
Status: built (self-test passing) · Targets: Claude Code · Codex · Cursor
About these two files. README.md is the AI-optimized source of truth; this README.html is the human view generated from it. Convention: .md = AI optimization, .html = human view.

The problem it solves

LUMEN's feature catalog scored 9/9 on uniformity because all 20 dossiers share the exact same 8 H2 headings in the same order, slugs are predictable, and numbering is gap-free. Authoring by hand invited the drift the benchmark found: three different metadata-header formats (16 / 2 / 2) and broken Related-features links. This skill removes the drift at the source — a generator that can only emit the canonical schema, with one metadata syntax, and that refuses to write a dossier that fails validation.

What it does — 6 functionalities

01 holds the schema and renders the .md; 02 derives slug + next number; 03 writes the .md; 04 generates the .html from it; 05 updates the index; 06 validates and gates --write. Each has its own doc under functionalities/.

01 Schema template

The canonical 8-section schema as the single source of truth; renders the dossier .md deterministically.

02 Slug & numbering

Kebab slug from the name + the next contiguous NN; reuses the number if the slug already exists.

03 Markdown generation writes

Writes NN-<slug>/<slug>.md — the AI source of truth — from the schema renderer.

04 HTML generation writes

Generates <slug>.html from the .md so constants never diverge; dark theme, self-contained.

05 Index & manifest update writes

Appends one catalog row to FEATURES.md (and a manifest entry if present); idempotent.

06 Validation gate

Asserts the 8 sections in order + one metadata syntax + no self-reference; a FAIL blocks --write.

The schema it produces

Ported verbatim from the real dossiers. # Title → One-line scope: → **Group:** → **Status:** → then the 8 H2 sections, in order:

#Section
1## Summary
2## What it does / mechanics
3## How it is built (implementation)
4## Data-contract touchpoints
5## Model B / rights notes
6## Key parameters / values
7## Related features
8## References

One metadata syntax only — strict markdown **Group:** / **Status:** (matching all real dossiers), never YAML frontmatter. This fixes the benchmark's header variance. The filename is the slug only (signal-taxonomy.md, not 03-signal-taxonomy.md).

Output

  • NN-<slug>/<slug>.md — the dossier on the canonical schema (AI source of truth).
  • NN-<slug>/<slug>.html — the human view, generated from the .md.
  • FEATURES.md — one catalog row added (if an index exists).
  • .feature-creator/backup/<run>/ — backups of any overwritten file.

Empty sections carry dense, instructive _TODO_ placeholders. Loop: scaffold → fill the .md _TODO_s → re-run to regenerate the .html.

Safety & determinism

  • Dry-run by default. No writes without --write; the plan shows create / update / unchanged.
  • Validated before write. A schema FAIL blocks --write.
  • Idempotent. Same name + catalog → existing number reused, second run writes nothing (no duplicate folder/row).
  • Backups. Every overwritten file is copied to .feature-creator/backup/<run>/ first.
  • Deterministic. No Math.random/Date in output content; same inputs → byte-identical output.

How to run

node tasks/skill-feature-structure-creator/scripts/create.mjs "My Feature" --features-dir lumen-features   # dry-run

node tasks/skill-feature-structure-creator/scripts/create.mjs "My Feature" --features-dir lumen-features \
  --group "Detection Engine" --status "Specified only" --scope "One-line scope here." --write          # apply

node tasks/skill-feature-structure-creator/scripts/self-test.mjs                                        # idempotency + schema gate
